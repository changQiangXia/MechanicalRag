**Mechanical Embodied RAG 项目 — 代码说明与解释**

本文档按照当前版本的实际代码结构，说明根目录脚本和 `simulation/` 模块的职责，便于继续修改和维护。

**一、根目录核心文件**

| 文件 | 作用 |
|------|------|
| `env_test.py` | 检查 `langchain`、`chromadb`、`transformers`、`sentence-transformers` 是否可用 |
| `download_models.py` | 通过 ModelScope 预下载模型 |
| `model_provider.py` | 统一处理 ModelScope 与 Hugging Face 的模型路径解析 |
| `chroma_compat.py` | 处理 Chroma 配置兼容问题 |
| `llm_loader.py` | 统一加载 Qwen2 或 Flan-T5，并封装成 LangChain 可调用对象 |
| `mechanical_data.txt` | 机械领域知识库 |
| `base_rag.py` | 基础 RAG，本地检查时可直接运行 |
| `improved_rag.py` | 改进版 RAG，当前采用结构化分条索引、意图扩展检索和抽取式回答 |
| `problem_solving.py` | 更规则化的问答链路，用来保留定向修复策略；当前 8 题上与 `improved_rag` 同分 |
| `qa_dataset.py` | 问答评测数据集与判分规则 |
| `rag_evaluation.py` | 四条问答链路的结构化评测入口 |
| `visualize_results.py` | 根据 JSON 结果生成图表 |
| `generate_showcase.py` | 生成展示摘要 `showcase_summary.txt` |
| `run_all.sh` | 一键复现主流程 |
| `requirements.txt` | 项目依赖 |

**二、问答相关代码**

`base_rag.py`

- 用 `TextLoader` 和 `RecursiveCharacterTextSplitter` 切分知识库
- 用 `HuggingFaceEmbeddings` 和 `Chroma` 建立向量库
- 用 `RetrievalQA` 形成最基础的 RAG 链路

`improved_rag.py`

- `_parse_entries`：把知识库按编号条目拆开，保留类别信息
- `INTENT_PROFILES`：定义典型机械问题的意图触发词、扩展查询和关注词
- `custom_retriever`：融合语义检索、词项命中、类别偏好
- `_extract_answer`：对夹爪力范围、标定复位、圆柱形姿态、薄壁件等问题做抽取式回答
- `answer_question`：优先用抽取结果，必要时再调用 LLM

`problem_solving.py`

- 保留更完整的定向规则与模板
- 在当前项目里主要用于记录问题修复方案，当前 8 题结果与 `improved_rag` 一致

`qa_dataset.py`

- 每题都包含 `required_groups`、`evidence_groups`、`forbidden_keywords`
- 这样可以把回答错误和检索错误区分开

`rag_evaluation.py`

- 同时评估 `direct_llm`、`base_rag`、`improved_rag`、`problem_solving_rag`
- 其中 `direct_llm` 不经过检索，主要用来补齐 baseline
- 输出 `qa_evaluation_detail.json`、`rag_evaluation_report.txt`、`rag_problems.txt`

**三、仿真相关代码**

| 文件 | 作用 |
|------|------|
| `simulation/tasks.py` | 定义 benchmark 任务、理想夹爪力范围、推荐接近高度、任务划分 |
| `simulation/env.py` | 负责执行 pick-and-place，返回 success、耗时、步数、距离误差、力偏差等信息 |
| `simulation/rag_controller.py` | 将任务描述映射成控制参数 |
| `simulation/baseline_controller.py` | 提供 `fixed`、`random`、`direct_llm` 等基线参数 |
| `simulation/feedback.py` | 失败后参数调整逻辑 |
| `simulation/seed_utils.py` | 生成跨进程稳定种子偏移 |
| `simulation/benchmark.py` | 单方法 benchmark、多方法对比、多 seed 汇总、消融实验的主入口 |
| `simulation/learned_controller.py` | 学习型参数模块 |
| `simulation/train_learned_model.py` | 单独训练学习型模块 |
| `simulation/scene.xml` | MuJoCo 场景文件 |

**四、benchmark 代码重点**

`simulation/benchmark.py` 当前最重要的几个函数：

- `run_benchmark`：单种方法、多任务、多次试验
- `run_benchmark_multi_seed_report`：为 `simulation_benchmark_result.json` 生成多 seed 汇总结果
- `run_benchmark_comparison`：生成 `rag / direct_llm / fixed` 单 seed 对比
- `run_benchmark_comparison_multi_seed`：生成三种方法的多 seed 汇总
- `run_retrieval_ablation`：检索策略消融

CLI 常用入口：

```bash
python -m simulation.benchmark --report_multi_seed --method rag --n_trials 20 --seeds 42 43 44 --output simulation_benchmark_result.json
python -m simulation.benchmark --compare_direct_llm --n_trials 20
python -m simulation.benchmark --compare_multi_seed --n_trials 20 --seeds 42 43 44 --multi_seed_methods rag direct_llm fixed
```

**五、图表与展示代码**

`visualize_results.py`

- 读取 `qa_evaluation_detail.json`
- 读取 `simulation_comparison_rag_vs_baseline.json`
- 读取 `simulation_comparison_multi_seed.json`
- 当前生成 15 张图，覆盖问答结果、对比增益、力偏差、接近高度、split 汇总、多 seed 成功率等

`generate_showcase.py`

- 读取问答、仿真和 benchmark JSON
- 自动抽取代表性问答样例
- 汇总多 seed 成功率增益
- 输出 `showcase_summary.txt`

**六、当前主流程**

当前推荐的主流程在 `run_all.sh` 中已经固定：

1. `env_test.py`
2. `rag_evaluation.py`
3. `problem_solving.py`
4. `simulation.benchmark --report_multi_seed`
5. `simulation.benchmark --compare_direct_llm`
6. `simulation.benchmark --compare_multi_seed`
7. `visualize_results.py`
8. `generate_showcase.py`

`base_rag.py` 和 `improved_rag.py` 还在仓库中保留，主要用于局部检查，不再是主复现流程的一部分。

**七、结果文件如何对应代码**

| 结果文件 | 来源 |
|------|------|
| `qa_evaluation_detail.json` | `rag_evaluation.py` |
| `rag_evaluation_report.txt` | `rag_evaluation.py` |
| `rag_problems.txt` | `rag_evaluation.py` |
| `problem_solving_result.txt` | `problem_solving.py` |
| `simulation_benchmark_result.json` | `simulation.benchmark --report_multi_seed` |
| `simulation_comparison_rag_vs_baseline.json` | `simulation.benchmark --compare_direct_llm` |
| `simulation_comparison_multi_seed.json` | `simulation.benchmark --compare_multi_seed` |
| `showcase_summary.txt` | `generate_showcase.py` |

**八、后续改代码时最常用的入口**

- 改问答数据集：`qa_dataset.py`
- 改改进版检索：`improved_rag.py`
- 改问题解决型规则：`problem_solving.py`
- 改仿真任务难度：`simulation/tasks.py`
- 改仿真成功逻辑：`simulation/env.py`
- 改 benchmark 输出格式：`simulation/benchmark.py`
- 改图表：`visualize_results.py`
