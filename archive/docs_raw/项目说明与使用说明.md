**Mechanical Embodied RAG 项目 — 交付说明与使用说明**

这份文档用于日常复现、交付和答辩展示，重点说明项目能做什么、怎么运行、会生成什么结果。

**一、项目简介**

项目面向机械工程与具身智能场景，核心内容有两部分：

- 问答链路：三条知识库驱动的问答链路，加上一条 `direct_llm` baseline
- 仿真链路：用知识库驱动抓取参数选择，并在仿真 benchmark 中验证效果

当前版本已经包含：

- `direct_llm`、`base_rag`、`improved_rag`、`problem_solving_rag`
- `rag`、`direct_llm`、`fixed`
- 多 seed 统计
- 图表生成
- 展示摘要

其中 `direct_llm` 直接走模型回答，用来补齐 baseline；其余三条问答链路会读取 `mechanical_data.txt`。

**二、环境安装**

建议在项目根目录执行：

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

模型默认通过 ModelScope 下载：

```bash
python download_models.py
```

如果需要启用 MuJoCo 仿真：

```bash
pip install mujoco
```

**三、如何运行**

完整复现：

```bash
bash run_all.sh
```

手动执行核心命令：

```bash
source venv/bin/activate
python rag_evaluation.py --data_path mechanical_data.txt
python problem_solving.py --data_path mechanical_data.txt
python -m simulation.benchmark --report_multi_seed --method rag --n_trials 20 --seeds 42 43 44 --output simulation_benchmark_result.json
python -m simulation.benchmark --compare_direct_llm --n_trials 20
python -m simulation.benchmark --compare_multi_seed --n_trials 20 --seeds 42 43 44 --multi_seed_methods rag direct_llm fixed
python visualize_results.py --qa_json qa_evaluation_detail.json --sim_json simulation_comparison_rag_vs_baseline.json --sim_multi_seed_json simulation_comparison_multi_seed.json --output_dir visualizations
python generate_showcase.py --qa_json qa_evaluation_detail.json --sim_json simulation_comparison_rag_vs_baseline.json --sim_multi_seed_json simulation_comparison_multi_seed.json --sim_benchmark_json simulation_benchmark_result.json --output showcase_summary.txt
```

**四、交付物清单**

运行完成后，根目录会生成以下主要文件：

| 文件 | 内容 |
|------|------|
| `qa_evaluation_detail.json` | 问答结构化结果 |
| `rag_evaluation_report.txt` | 问答文本报告 |
| `rag_problems.txt` | 问答问题清单 |
| `direct_llm_result.txt` | 直接 LLM 的逐题回答 |
| `problem_solving_result.txt` | 问题解决型 RAG 的逐题回答 |
| `simulation_benchmark_result.json` | `rag` 的多 seed benchmark |
| `simulation_comparison_rag_vs_baseline.json` | 单 seed 对比 |
| `simulation_comparison_multi_seed.json` | 多 seed 对比 |
| `showcase_summary.txt` | 展示摘要 |
| `visualizations/` | 图表文件 |

**五、当前结果摘要**

问答方面：

- `direct_llm`：0.25 / 0.375
- `base_rag`：0.25 / 0.375
- `improved_rag`：1.00 / 1.00
- `problem_solving_rag`：1.00 / 1.00

仿真方面，`rag` 的多 seed benchmark 成功率为：

- `pick_smooth_metal`：0.8167
- `pick_rubber`：0.8667
- `pick_small_part`：0.8500
- `pick_large_part`：0.7667
- `pick_thin_wall`：0.7833
- `pick_metal_heavy`：0.7500

从多 seed 对比结果看，`rag` 在 `pick_rubber`、`pick_small_part`、`pick_thin_wall` 上相对 `direct_llm` 的成功率增益分别约为 `0.7167`、`0.7667`、`0.7000`，这三类任务的区分度更清楚。

**六、补充说明**

- `base_rag.py` 和 `improved_rag.py` 仍可单独运行，适合做局部检查
- `simulation_benchmark_result.json` 当前已经是多 seed 汇总格式，适合直接放入汇报
