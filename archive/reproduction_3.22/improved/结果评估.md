**一、结果概览**

这一轮整理后，问答链路、仿真链路和展示材料都已经更新到同一套结果上。

- 问答部分保留了 `direct_llm`、`base_rag`、`improved_rag`、`problem_solving_rag` 四条链路
- 仿真部分保留了 `rag`、`direct_llm`、`fixed` 三组方法，并统一给出多 seed 汇总
- 可视化目录当前包含 15 张图，问答差异、仿真差异、稳定性和任务分布都能覆盖到
- `showcase_summary.txt` 已经加入主流程，汇报时可以直接引用

**二、问答结果**

当前问答评测结果如下。

- `direct_llm`：strict accuracy 0.25，weighted accuracy 0.375
- `base_rag`：strict accuracy 0.25，weighted accuracy 0.375
- `improved_rag`：strict accuracy 1.00，weighted accuracy 1.00
- `problem_solving_rag`：strict accuracy 1.00，weighted accuracy 1.00

这组结果说明了三点。

- 直接 LLM 在机械细节题上仍然容易偏离知识库，圆柱形工件姿态题、光滑金属夹爪力题、橡胶夹爪力题和薄壁件注意事项题都出现了明显错误
- 基础 RAG 的检索覆盖率有所提升，回答阶段仍然会出现整段照搬、跨材料范围混写、检索片段错位等问题
- 改进版 RAG 通过结构化分条索引、混合检索和抽取式回答，已经和基础 RAG 拉开了稳定差距

从链路关系来看，这一轮已经补齐了直接 LLM baseline，也把基础 RAG、改进版 RAG、问题解决型 RAG 的层次关系补完整了。

**三、仿真结果**

`simulation_benchmark_result.json` 已经从单 seed 结果改成多 seed 汇总结果，当前记录的是 `rag` 在 3 个 seed、每个任务 20 次试验下的 mean±std。

- `pick_smooth_metal`：0.8167 ± 0.0289
- `pick_rubber`：0.8667 ± 0.0764
- `pick_small_part`：0.8500 ± 0.1803
- `pick_large_part`：0.7667 ± 0.0764
- `pick_thin_wall`：0.7833 ± 0.0764
- `pick_metal_heavy`：0.7500 ± 0.0500

这组数据可以直接放进报告，原因有三点。

- 字段里已经包含 `task_id`、`task_split`、`ideal_force_center`、`ideal_approach_height`、`success_rate_mean`、`success_rate_std`
- 同一个任务在不同 seed 下的结果也保留在 `per_seed_success_rate` 中，后续做补充说明会方便很多
- `pick_metal_heavy` 的中文描述已经改成抓取重型光滑金属零件，和 `pick_smooth_metal` 能区分开

`simulation_comparison_multi_seed.json` 中，三种方法的对比如下。

- `pick_smooth_metal`：RAG 0.8167，Direct LLM 0.6000，Fixed 0.2500
- `pick_rubber`：RAG 0.8667，Direct LLM 0.1500，Fixed 0.2500
- `pick_small_part`：RAG 0.8500，Direct LLM 0.0833，Fixed 0.2167
- `pick_large_part`：RAG 0.7667，Direct LLM 0.4333，Fixed 0.4167
- `pick_thin_wall`：RAG 0.7833，Direct LLM 0.0833，Fixed 0.0833
- `pick_metal_heavy`：RAG 0.7500，Direct LLM 0.4500，Fixed 0.3333

这一轮里，RAG 相对 Direct LLM 的平均成功率增益约为 0.5056，相对固定基线的平均成功率增益约为 0.5472。图表里已经能清楚看到这些差异。

**四、本轮解决的问题**

这次重点处理了之前评估里列出的三项遗留问题。

- `improved_rag` 没有稳定优势：已经通过结构化索引、意图扩展检索和抽取式回答修复，当前问答结果从 0.375 提升到 1.00
- `simulation_benchmark_result.json` 仍是单 seed 结果：已经改成多 seed 汇总报告，正式汇报可以直接引用该文件
- 基础 RAG 和改进版 RAG 的演示入口偏弱：`run_all.sh` 已改成优先生成结构化评测、可视化和 `showcase_summary.txt`，展示链路更完整

除此之外，还补了一个读表时容易忽略的小问题。

- `pick_metal_heavy` 之前和 `pick_smooth_metal` 的中文描述重复，现在已经改成抓取重型光滑金属零件

**五、最适合汇报的结论**

- 这套实验链路已经具备直接 LLM baseline，问答部分的对比关系更严谨
- 改进版 RAG 和问题解决型 RAG 在当前 8 道机械题上都取得了满分表现，说明知识库组织方式和答案约束策略对结果影响很大
- 仿真 benchmark 已经具备多任务、多 seed、时间差异、95% 区间和基线对比，结果完整度明显高于原始版本
- 当前最有说服力的仿真差异来自橡胶件、小件、薄壁件三类任务，这几项在图里区分度很高
